## Один к одному (One-to-One, 1:1)

 Каждая запись в таблице A связана не более чем с одной записью в таблице B, и наоборот.


### Пример:

teachers

| id  | surname |
| --- | ------- |
| 1   | Петров  |
| 2   | Иванов  |
| 3   | Сидоров |

lessons

| id  | title          | teacher_id |
| --- | -------------- | ---------- |
| 1   | математика     | 3          |
| 2   | информатика    | 2          |
| 3   | русский        | 1          |


## Один ко многим (One-to-Many, 1:N)

 Одна запись в таблице A может быть связана с несколькими записями в таблице B, но запись в B ссылается только на одну запись в A.

### Пример:

teachers

| id  | surname |
| --- | ------- |
| 1   | Петров  |
| 2   | Иванов  |
| 3   | Сидоров |

lessons

| id  | title          | teacher_id |
| --- | -------------- | ---------- |
| 1   | математика     | 1          |
| 2   | информатика    | 1          |
| 3   | русский        | 2          |
| 4   | литература     | 2          |
| 5   | физра          | 3          |


## Многие ко многим (Many-to-Many, M:N)

 Одна запись в таблице A может быть связана с несколькими записями в таблице B, и наоборот.

### Пример:


teachers

| id  | surname |
| --- | ------- |
| 1   | Петров  |
| 2   | Иванов  |
| 3   | Сидоров |

lessons

| id  | title          | teacher_id |
| --- | -------------- | ---------- |
| 1   | математика     | 1          |
| 1   | математика     | 3          |
| 2   | информатика    | 1          |
| 3   | русский        | 2          |
| 3   | русский        | 3          |
| 4   | литература     | 2          |
